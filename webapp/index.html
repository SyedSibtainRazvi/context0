Local-first context handoff

Keep coding context when you switch tools.

switch saves a checkpoint for your current repo and branch, then resumes it in Claude Code, Cursor, Codex, or plain terminal.

Start in 2 minutes Use with MCP

What is switch?

AI coding sessions do not share memory across tools. switch stores compact checkpoints in local SQLite so your "what I did" and "what to do next" follow your git branch.

Scoped

Context key is repo_root + branch, so each branch stays isolated.

Local

No cloud dependency. Data is saved at ~/.switch/switch.db by default.

Fast

Save, resume, and log are lightweight CLI operations built for active dev loops.

Quickstart

Best flow: install switch, add MCP + a rule file, then let the AI save and resume context. Say “save context” or “I’m switching” when you change tools.

  1. Install

    cargo install --path .

  2. Add MCP and a rule file

    Configure switch mcp-server in your tool (see MCP setup below). Then add the rule so the agent uses switch: Cursor → rules/switch.mdc into .cursor/rules/; Claude Code → rules/CLAUDE.md; Codex → rules/AGENTS.md.

  3. Use it

    When you open a session, the agent calls get_context and resumes. When you say “save context” or “I’m switching”, the agent calls save_context with a summary. No manual CLI needed.

  4. CLI fallback

    switch save --done "wired auth middleware" --next "fix integration tests"

    Resume: switch resume or switch resume --json

Commands

save

Write a checkpoint for the current repo + branch.

switch save \
  --done "implemented OAuth flow" \
  --next "add refresh token logic" \
  --blockers "waiting on API key" \
  --tests "cargo test auth::" \
  --files src/auth.rs src/middleware.rs

resume

Print the latest checkpoint for the current repo + branch.

switch resume
switch resume --json

log

Show recent checkpoints in reverse chronological order.

switch log --limit 20

mcp-server

Run MCP over stdio so tools can read/write context directly.

switch mcp-server

MCP setup

Add switch mcp-server to your editor/tool MCP config and use tools: get_context, save_context, and list_context. 

{
  "mcpServers": {
    "switch": {
      "command": "switch",
      "args": ["mcp-server"]
    }
  }
}

File: ~/.claude/settings.json

{
  "mcpServers": {
    "switch": {
      "command": "switch",
      "args": ["mcp-server"]
    }
  }
}

File: .cursor/mcp.json (project) or ~/.cursor/mcp.json (global)

Storage and scope

Default database

~/.switch/switch.db

Custom database

switch --db /path/to/custom.db resume
